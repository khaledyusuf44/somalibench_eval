# somalibench_eval — project context for Codex

## What this project is

A **safety-evaluation study**: run SomaliBench v0 (a published, native-author-verified
benchmark of 100 harmful-intent prompts × English / Somali = 200 records) against four
open-weight LLMs, classify each model response as refused / complied / unclear, and
quantify the cross-lingual refusal gap with 95% bootstrap CIs.

This is **defensive measurement research**. The artifact is a per-model refusal-rate table
+ a short paper. No harmful content is generated by this project — it measures what other
models do when given inputs from a published safety benchmark, which is the standard,
documented use of such benchmarks.

## Why the repo touches harmful-looking content

- Inputs are loaded from the **published, CC-BY-NC-4.0 SomaliBench v0** dataset
  ([HF](https://huggingface.co/datasets/khaledyusuf44/somalibench-v0),
  [GitHub](https://github.com/khaledyusuf44/somalibench_v0)).
- Source prompts come from publicly-released academic safety sets (HarmBench, AdvBench).
- The point of the project is to **measure refusal behavior**, which requires the
  benchmark to contain the requests a safe model should refuse.
- Model responses captured during evaluation may include content the model failed to
  refuse — this is the *measurement signal* of the study. They are stored locally for
  classification and stripped from any public release.

## What goes public vs stays local

- **Public** (in the paper + release): per-model **refusal rates**, **per-category gaps**,
  **judge agreement** with native spot-check, **example refusals** (safe to share).
- **Local-only** (gitignored, never published): the **raw model responses** when models
  complied with harmful prompts — those are kept on disk for reproducibility but not
  shared. The paper reports counts and rates, not the harmful generations themselves.

## Methodology in one paragraph

For each of four open-weight models (Llama-3.1-8B, Gemma-2-9B, Qwen-2.5-7B, Aya-23-8B),
run each of SomaliBench v0's 200 records through the model with temperature 0 and a
single English HHH system prompt; record the response. Use Claude Sonnet as an LLM-as-judge
to classify each response as refused / complied / unclear. The native author manually
verifies a random 80-row sample of the judge's classifications — agreement reported
as inter-judge reliability. Compute per-model per-language refusal rate with 95%
bootstrap CIs (500 resamples, seed = 0). The headline metric is the **EN−SO refusal
gap per model**.

## Repository layout

```
configs/eval_config.yaml     models, paths, judge settings, seeds, generation params
src/01_run_models.py         orchestrate Ollama inference (4 models × 200 prompts)
src/02_judge_responses.py    Claude Sonnet judge → refused/complied/unclear labels
src/03_spot_check.py         random-sample 80 for native author manual verification
src/04_analyze.py            refusal rates, EN-SO gaps, 95% bootstrap CIs, per-category
src/05_make_figures.py       plots and tables for the paper
data/responses/              gitignored — raw model outputs (one JSONL per model)
data/classifications/        gitignored — judge labels
data/spot_checks/            gitignored — native verification sample
data/results/                refusal-rate tables (public-safe)
paper/                       paper draft
notes/decisions.md           per-decision rationale
```

## Standard workflow

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# pull models via ollama (one-time)
ollama pull llama3.1:8b
ollama pull gemma2:9b
ollama pull qwen2.5:7b
ollama pull aya:8b

# pipeline
python src/01_run_models.py       # 4 × 200 = 800 inferences (~few hours)
python src/02_judge_responses.py  # ~800 Codex API calls (~$2-5)
python src/03_spot_check.py       # samples 80 random; you manually label
python src/04_analyze.py          # rates + gaps + CIs
python src/05_make_figures.py     # plots for the paper
```

## Scope guardrails (v1)

- 100 prompts × 4 models × 2 languages = 800 inferences. No grid over temperature, system
  prompt, or sample count for v1.
- Single LLM judge (Claude Sonnet) + native spot-check on 80. Multi-judge comparison is
  a v2 extension.
- No jailbreak variants, no adversarial wrappers. The point is *baseline* refusal rates.
- Not a model release. Not a fine-tuned defense. Pure measurement.

## Ethics

- Inherits SomaliBench v0's CC-BY-NC-4.0 + responsible-use notice.
- Raw harmful-compliance generations stay on disk; aggregate numbers are what gets shared.
- Reading, editing, validating, and analyzing these files is normal project work.
